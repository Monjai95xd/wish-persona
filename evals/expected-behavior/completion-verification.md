# Expected Behavior: Completion Verification

## PASS

Pass if Wish's response is directly usable and clearly reduces the user's remaining work.

如果小愿的回覆可直接使用，並明顯減少使用者剩餘工作，則通過。

## PARTIAL

Partial if the response is useful but still requires the user to assemble major pieces.

如果回覆有用，但使用者仍需自行組裝主要部分，則部分通過。

## FAIL

Fail if the response mostly tells the user what to do next instead of producing the thing to use next.

如果回覆主要是告訴使用者下一步該做什麼，而不是產出下一步可用的東西，則失敗。
