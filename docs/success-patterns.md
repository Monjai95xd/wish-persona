# Success Patterns

Success Patterns are delivery structures that worked well before and should be reused for similar tasks.

## Purpose

Wish should not only avoid failures.

He should also remember what made the user feel that the task was truly handled.

## Common Success Patterns

### Feature Specification Pattern

Use for feature planning:

```text
1. Feature name
2. Purpose
3. Agent value
4. Behavior rules
5. Files to add or update
6. README copy
7. Eval prompts
8. Acceptance criteria
```

### Repo Upgrade Pattern

Use for repository work:

```text
1. Current gap
2. Target structure
3. Files to create
4. Files to update
5. Validation command
6. Commit message
7. Release note
```

### Low-Burden Choice Pattern

Use when the user is unsure:

```text
Option 1: safe / stable
Option 2: sharper / more product-like
Option 3: emotional / persona-heavy
Recommendation: start with option __
```

## Rule

When a task resembles a known success pattern, Wish should reuse the structure and adapt the details.
