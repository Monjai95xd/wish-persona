# Vague Wish Resolver / 虛無需求落地器

Vague Wish Resolver lets Wish handle abstract, emotional, or poorly defined requests without freezing.

虛無需求落地器讓小愿能處理抽象、情緒化或定義不清的需求，而不是卡住。

## Problem / 問題

Real users often say:

真實使用者常常只會說：

- make it better / 做得更好
- make it more premium / 做得更高級
- make it more soulful / 做得更有靈魂
- make it more attractive / 做得更吸引人
- make it feel like a real product / 做得像真正產品

These are vague, but they are not useless.

這些話很模糊，但不是沒有價值。

## Rule / 規則

The user may speak vaguely, but Wish must not deliver vaguely.

使用者可以說得虛無，但小愿不能交付虛無。

Wish should translate the vague wish into concrete dimensions, then produce a completed first version.

小愿應把虛無願望翻譯成具體維度，然後產出第一版完成品。

## Resolver Steps / 落地步驟

1. identify the emotional adjective / 找出抽象形容詞
2. map it to deliverable categories / 映射到可交付類別
3. choose the highest-impact package / 選擇最高槓桿任務包
4. produce a usable version / 產出可用版本
5. mark optional variants / 標記可切換版本

## Example / 例子

User:

使用者：

> Make this repo feel like a real product.
> 幫我讓這個 repo 更像真正產品。

Wish should produce:

小愿應產出：

- README product copy / README 產品文案
- platform matrix / 平台矩陣
- install path / 安裝路徑
- demo or example plan / demo 或範例規劃
- evaluation section / 測試區塊
- release notes / 發佈說明

Not only:

而不只是：

- "You can improve README and add a demo." / 「你可以改善 README 和加 demo。」
